# 需求文档输出技能

本技能指导 agent 使用标准模板将需求分析结果输出为结构化的 `spec.md` 文档。

---

## 工作流程

1. **读取模板**：读取 `templates/spec-template.md`，理解其结构和占位符。
2. **填写模板**：根据需求分析结果，逐段填写模板。参照下文的填写指南。
3. **确定输出路径**：与用户确认输出路径，默认建议 `specs/<feature>/spec.md`。
4. **写入文件**：将填写后的内容写入确认的输出路径。

---

## 模板填写指南

模板文件：`templates/spec-template.md`

- **关联 Issue**：必须链接到对应的 GitHub Issue
- **概述**：说明业务背景、目标用户画像、核心价值主张和成功标准
- **干系人与用户**：列出所有相关方及其角色和利益诉求
- **需求范围**：明确"做"与"不做"，定义边界条件和外部依赖
- **功能需求**：按 MoSCoW（Must/Should/Could/Won't）优先级排列，包含用户故事、验收标准和边缘情况，每个需求须可测试
- **非功能需求**：性能指标、安全约束、可用性、可维护性、可扩展性等
- **验收标准**：每条须有明确的通过/失败条件
- **依赖与假设**：标注外部依赖和已做的假设（便于后续验证）
- **风险与开放问题**：记录已知风险和待回答的问题
- **待澄清事项**：记录所有未决问题和需要客户确认的假设
- **修订记录**：每次变更记录版本号、日期、变更内容和变更人

---

## 输出路径

文档输出前，须与用户确认输出路径，默认建议 `specs/<feature>/spec.md`（遵循项目 `AGENTS.md` 中 `specs/` 目录约定）。

---

## 语言

所有输出使用中文。用户故事、验收标准等结构字段可使用中英文混排（如 As a / I want / So that 及 Given-When-Then），以团队约定为准。
