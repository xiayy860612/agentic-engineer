# 测试类型与适用场景

> 本文档是根 `AGENTS.md`「测试先行」章节的补充，列出各测试类型及其适用场景。
> 具体框架、命令与覆盖率配置见各子项目 `AGENTS.md`。

## 测试类型

| 方式 | 适用场景与要求 |
|------|----------------|
| **单元测试** | 隔离依赖，覆盖领域逻辑、纯函数、分支与边界；单元测试覆盖率门禁见根 `AGENTS.md` Quality gates。 |
| **集成测试** | 多模块协作及与数据库、消息队列、外部 HTTP 等边界的真实或容器化联调。 |
| **契约测试（含消费者驱动契约）** | 多方约定请求/响应与版本；契约变更须**先**改契约与失败用例，再改实现与消费者。 |
| **组件 / UI 测试** | 组件行为、关键交互与可访问性（具体库由子项目定）。 |
| **端到端（E2E）测试** | 关键用户旅程；不替代应有的单元覆盖，避免慢、脆的堆砌。 |
| **快照测试** | 稳定输出对比；须人工审阅 diff，禁止无意义整批接受。 |
| **属性测试** | 复杂不变量或大量输入组合；在子项目栈支持时用于补强单元测试。 |

## 豁免

**纯文档**、**纯配置重排**、**仅重命名且无行为变化**须在 PR 或说明中**一句话声明理由**；其余情形不得豁免。覆盖率等其他门禁的例外仅按根 `AGENTS.md` Quality gates 办理。
